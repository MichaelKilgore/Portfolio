Favorite Projects

Shopping List App

non-technical description

The Shopping Lists App is an IOS application that allows users to easily create and share lists. List items contain text, an optional link, and an optional image. The Shopping Lists App beats out competition with its real time updates of lists, efficiently designed RESTful service, and beautifully designed user interface.

technical elaboration

This application uses NodeJS to run the REST service that helps users make updates to their lists, and automatically updates users of list changes with Pushkit. The service is run on AWS EC2, information is stored in MongoDB, and photos are stored in S3. The front end of the application is built using Swift.

Highlights
  • 2-step email authentication is used to for added protection of users information
  • The front-end is built with the latest and greatest SwiftUI which now is built with an MVVM software architectural pattern.
  • The server is built on NodeJS which will allow me to make the application support cross-platform communication in the future. This means to make the app available for android device users, I would only need to program the front end for android devices.
  • To reduce load on the server, and improve user experience, recently accessed photos are stored locally on the users device using Apples Core Data.
  • The entire backend is built on Promises and awaits. This eliminates all use of call backs making the code significantly more readable, and easier to debug.

Text Based Omni Games

non-technical description

Text Based Omni Games is a 4-player multiplayer game. It is a series of minigames run entirely in the terminal. Players compete in 4 different minigames all inspired by a minigame in the game Dota 2.

technical elaboration

The application was created entirely with C++, and users recieve updates on the game in real time from the host of game. The entire networking of the applicationis built entirely by me from the application layer, no additional libraries or APIs were used to handle client-server communications.

Highlights
  • The server is implemented using multi-threading to handle user input as well as communications with users in the game.
  • The client is also multi-threaded and simultaneously handles user interactions with the text-based UI and recieves asynchronous updates from the server on the game, and other players state.
  • Networking code was written seperately from the game logic allowing for reuse of the networking code on future projects.

Pet Simulator

non-technical description

Pet Simulator was a game where your pet would run around on the map and you could drop food, water, and toys to keep your pet happy and alive. The goal was to keep the pet alive. Your pet could die from boredom, starvation, and thirst.

technical elaboration

Pet Simulator was built entirely using C++ and a 2d graphing API called SFML. Follows object oriented designs, and makes use of inheritence.

Highlights
  • Worked on a team of 3 members who I met the day of the hackathon.
  • Went from conception, to mapping out the design of the program, to dividing up the work, and ultimately finishing our game in under 24 hours.
